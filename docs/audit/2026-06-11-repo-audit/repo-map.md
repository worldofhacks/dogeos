## Frontend (apps/web)

The DogeSwap frontend (/Users/quietguy/Documents/Dev/dogeos/apps/web/src) is a React 18 + Vite 6 app written in plain JSX (no TypeScript), styled entirely with inline styles over a hand-rolled theme-token system. It runs as TWO independent React roots: the main UI at #root (main.jsx -> ui/App.jsx -> ui/Shell.jsx -> four views) and a headless, lazily-mounted wallet bridge at #sdk-wallet-root (sdk-wallet.jsx) that publishes a window.dogeosAggregatorWallet global plus window CustomEvents. The Shell renders SwapView (the centerpiece), TokensView, ActivityView, SettingsView; SwapView owns the live quote (useQuote: debounced/polled/seq-guarded POST /quote), live balances (useTokenBalances: raw eth_call balanceOf), and spawns the SwapFlow modal which drives review -> /approval -> /swap -> receipt polling via useSwapExecution over lib/execute.js. Wallet connection is dual-mode: with a DOGEOS_CLIENT_ID the @dogeos/dogeos-sdk Connect Kit modal (sdk-wallet-provider.jsx) is the single chooser; without one (the default) a hand-written EIP-6963 injected bridge (injected-wallet.js, 609 lines) discovers providers, prefers MyDoge, and enforces chain 6281971 via wallet_switchEthereumChain/wallet_addEthereumChain. State management is deliberately minimal: useState + custom hooks, two contexts (SettingsCtx persisted to localStorage 'doge.settings', ThemeCtx), and window events/globals for everything crossing the two React roots. The dev server embeds the entire aggregator backend in-process via a Vite middleware plugin (vite.config.mjs dogeosApiPlugin -> packages/api/src/live.mjs).

## Backend (packages/aggregator, api, web, config, dogeos-rpc)

The Node backend is a zero-framework, plain-ESM (.mjs) stack of five package folders sharing one root package.json. packages/api exposes a WHATWG-fetch-style handler (handler.mjs) over node:http (server.mjs); live.mjs is the composition root that wires the handler to packages/aggregator (quoting, routing, fee scoring, calldata building, on-chain verification) via a hand-rolled JSON-RPC client (packages/dogeos-rpc) pointed at the DogeOS Chikyu testnet RPC from packages/config/src/chains.mjs. packages/web/src/server.mjs is the production server: static dist + the same API handler + runtime config injection. Everything is dependency-injected factories with BigInt arithmetic, hand-rolled ABI encoding (no viem/ethers), a hardcoded venue registry of 3 ACTIVE + 2 WATCHLIST DEXes, and a fail-soft quote path / fail-closed swap path. The server never holds keys: /swap returns an unsigned transaction for the browser wallet to sign.

- Dependency injection everywhere: createX({deps}) factories returning closures, with every effect (fetchFn, nowMs, client, providers, timeouts) injectable and defaults wired only in api/src/live.mjs — the pure handler (handler.mjs) is fully testable offline
- BigInt for all monetary/gas values; JSON responses serialize bigint→string via a replacer (handler.mjs:20-29); all external inputs re-validated with BigInt() coercion plus regex hex-shape assertions (addresses /^0x[0-9a-f]{40}$/, calldata, selectors) — the same small assert helpers are copy-pasted into ~8 modules rather than shared
- Hand-rolled ABI encoding/decoding (selector + 64-hex-char word padding) instead of viem/ethers — e.g. venueCalldataBuilders.mjs, erc20Approval.mjs, v2Pools.mjs, l1GasPriceOracle.mjs
- Fail-soft quoting / fail-closed execution: quote providers wrap every source in Promise.race timeouts (1.5s provider / 1s source) and convert errors into quoteDiagnostics surfaced in /quote telemetry (composite.mjs:27-52, v2Pools.mjs:132-152); the swap path throws on any mismatch and maps to 422
- Source status taxonomy watchlist/readOnly/simulationOnly/active/disabled (registry.mjs:11-17) used consistently across registry, routing rejection reasons, and verification ladder; registry reads always return structuredClone copies (registry.mjs:513-582)
- Batch-then-fallback RPC pattern: try batchCall/batchGetCode, silently fall back to sequential calls if the RPC frontend rejects JSON-RPC batches (repeated in jsonRpcClient consumers)
- Self-executing module entrypoints guarded by import.meta.url === pathToFileURL(process.argv[1]).href (server.mjs:106, web/server.mjs:194); HOST/PORT from env with localhost defaults
- Tests use node --test (root script runs packages/**/*.test.mjs); offline-by-default thanks to DI; no JS CI (only Foundry/Slither contracts CI)

**Surprises:**

- Selector verification is a substring search: hasSelector (verification/verifySource.mjs:1-5) checks bytecode.includes(selector hex) anywhere in the code — any contract whose bytecode happens to contain those 4 bytes (e.g. in data sections or other push constants) passes the 'selector present' check. It is one input among several gates, but for an audit it is a weak primitive feeding the 'active' classification
- The live verification snapshot (Blockscout + read checks, /venues //verification endpoints) is display-only: the actual /swap execution gate re-checks the STATIC registry constants (calldataRegistry.mjs:39-61 imports listSources()), not the live snapshot — a venue that started failing live checks would still build calldata until the registry source file is edited; mitigation is the per-swap eth_call simulation in verifySwapTransaction
- CORS access-control-allow-origin * on all routes including POST /swap and /approval (handler.mjs:11-16), no auth, no rate limiting, and no request-body size limit (server.mjs:25-32 buffers the whole body) on either server
- outputWeiPerFeeWei defaults to 1n in live mode (live.mjs:143): route scoring converts the DOGE-denominated fee into output-token units at a fixed 1:1 wei ratio regardless of actual token price — netOutput rankings between venues are fee-distorted for non-DOGE-priced outputs (all official tokens are 18 decimals, so no decimals mismatch, but prices differ)
- In live mode /swap ignores client-supplied price fields by re-quoting server-side (refreshSwapQuoteBeforeBuild=true, live.mjs:157; handler.mjs:283-335 keeps only recipient/deadline/sender) — good — but quote.deadline itself comes from the client with only a non-negativity check (venueCalldataBuilders.mjs encodeUint); buildSwapTx never validates deadline is in the future, relying on quote TTL plus the on-chain router's own deadline check
- Data-finality fee oracle failures silently return 0 (l1GasPriceOracle.mjs:91-94 fallbackFeeWei=0n, and live.mjs constructs the provider without onProviderError) — total fee and balance preflight can be understated without any diagnostic
- One-hop routes are quote previews only: composeOneHopCandidates pins status 'readOnly' (oneHop.mjs:46-47), so the aggregator can only EXECUTE single-venue direct swaps despite advertising one-hop quotes; one-hop also skips exactOutput entirely (oneHop.mjs:79)
- V2 quotes are computed locally from getReserves with a hardcoded feeBps=30 (v2Pools.mjs:196) instead of calling the venue's getAmountsOut — a venue with a non-0.3% fee would be mis-quoted (caught later only by swap simulation/slippage)
- The chain verifier memoizes success forever per process (live.mjs:23-45): if the RPC endpoint behind the URL ever changed chains mid-process, no re-check occurs (chain-status endpoint would show it, but quote/swap pre-verifiers would keep passing)
- The in-flight /quote dedupe map (handler.mjs:409-439) is process-global and keyed only on request params — concurrent identical requests from different users share one candidate fetch and have each other's source-error diagnostics replayed into their telemetry (telemetry.sourceErrors also exposes raw provider error messages to any caller)
- GET /activity is a thin proxy to Blockscout (handler.mjs:80-99) — the only route that calls an external HTTP API other than the RPC and verification Blockscout fetches
- The 'aggregator' public API is the 87-line barrel packages/aggregator/src/index.mjs (~45 exports spanning registry, intelligence, verification, fees, adapters, providers, routing, and swap builders), but api/src/live.mjs bypasses it and deep-imports 12 submodules directly, so the real coupling surface is wider than the barrel suggests

**Data flow:**

ON-CHAIN EXECUTION (DogeSwapRouter.sol): A caller invokes execute() with a byte-per-command program drawn from a fixed 7-entry, movement-only whitelist (src/libraries/Commands.sol: PERMIT2_PERMIT 0x00, PERMIT2_TRANSFER_FROM 0x01, V2_SWAP 0x02, V3_SWAP 0x03, ALGEBRA_SWAP 0x04, WRAP_NATIVE 0x05, UNWRAP_NATIVE 0x06). _dispatch is a flat if/else; any other byte reverts UnknownCommand. There is no CALL/DELEGATECALL/arbitrary-target command, and the only external targets are four constructor-set immutables (WDOGE, MUCHFI_V2_ROUTER 0xC653e7…, MUCHFI_V3_ROUTER 0x54f7D7…, BARKSWAP_ALGEBRA_ROUTER 0x77147f…) plus the hardcoded canonical Permit2 (Constants.PERMIT2 = 0x…78BA3). LEDGER: execute() builds an in-memory Ledger{tokens, entry, pulled, count} sized 2n+2 (no storage, linear scan); native entry is seeded address(this).balance - msg.value, and buyToken is snapshotted up front. Every amount is a balance delta (_delta = current - entry); _spend caps any explicit amount at the delta (InsufficientLedgerBalance) and resolves the CONTRACT_BALANCE sentinel (type(uint256).max) to the delta — so pre-existing/airdropped/stranded funds are structurally unspendable via execute (invariants I1/I5); only owner rescue() can move them. PERMIT2: users approve Permit2, never the router; _permit2Permit enforces p.spender == address(this) and both Permit2 calls hardcode msg.sender as owner/from, so a third party's live allowance cannot be drained (THREAT_MODEL.md threat 2, RouterPermit2.t.sol). AGGREGATE CAP: _accrueInput sums per-token input across all PERMIT2_TRANSFER_FROM and WRAP_NATIVE commands in one execute and reverts NotionalCapExceeded above _capOf(token), which resolves per-token cap -> defaultMaxInputPerTx -> uncapped (type(uint256).max = explicit-uncapped sentinel) — launch blast-radius control (I8). SETTLEMENT: after the command loop, _settle (not any command) takes the fee (feeBps <= MAX_FEE_BPS = 100 = 1%, default 0), reverts MinOutNotMet if the measured buyToken delta after fee < s.minOut, pays s.recipient, and refunds every leftover input delta to msg.sender — venue return values are deliberately ignored (FoT-token safe; Slither unused-return triaged as FP). Reentrancy is blocked by OZ ReentrancyGuardTransient (EIP-1153 — Prague support probe-confirmed on DogeOS per audit/CHAIN_FACTS.md §3); receive() rejects everything except WDOGE unwrap proceeds. GOVERNANCE: Ownable2Step owner = (intended) OZ TimelockController whose proposer/executor/admin is the founder Safe; guardian is a pause-only hot key (pause: guardian|owner; unpause/setFee/caps/rescue: owner only) — matches the solo-founder Safe+timelock+pause-only-guardian intent and is end-to-end tested in test/RouterGovernance.t.sol with a real TimelockController and 2-day delay. DEPLOY (script/DeployRouter.s.sol + audit/DEPLOYMENT.md): single broadcast (1) deploys canonical Permit2 via the Arachnid CREATE2 proxy 0x4e59b4… ONLY if absent — Permit2 is confirmed ABSENT on DogeOS testnet (CHAIN_FACTS.md §4) and MUST land at the hardcoded address using the vanity salt 0x…d3af2663da51c10215000000 (salt 0 yields a different address; the script asserts computed == canonical); (2) deploys the TimelockController (delay from TIMELOCK_MIN_DELAY env, Safe as proposer/executor/admin); (3) deploys the router with the deployer as temporary owner; (4) sets defaultMaxInputPerTx + per-token caps (CAP_DEFAULT/CAP_WDOGE/CAP_USDC[/CAP_USDT]) and asserts feeBps()==0 in the same broadcast so the router is never live-and-uncapped; (5) deploys the registry, points it at the router, and transfers it to ROUTER_SAFE; (6) begins the Ownable2Step handover of the router to the timelock. Two manual post-deploy acceptances remain (timelock.acceptOwnership on the router via Safe schedule/execute; Safe acceptOwnership on the registry). Status per DEPLOYMENT.md: fork-validated, no real broadcast (no funded key). CI (.github/workflows/contracts-security.yml): two jobs on packages/contracts/** changes only — forge build --sizes + forge test -vvv with FOUNDRY_INVARIANT_RUNS=512/DEPTH=200, and crytic/slither-action with packages/contracts/slither.config.json (fail_on: high).
